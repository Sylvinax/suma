Frequently Asked Questions

Here are some additional concepts, best practices and troubleshooting tips to help you make the most of this CollabNet product.

  • Do I need to have Mylyn installed to use the CollabNet Desktop plugins for Project Tracker or Issue Tracker?
    No, the CollabNet Desktop tracker plugins do require Eclipse Mylyn, but the installer installs the correct version of Eclipse Mylyn for you as part of the overall install process.
  • I've installed CollabNet Desktop but can't see any of the Mylyn features. What gives?
    Use the file menu and see if you can select Window > Show View > Other > Mylyn. That should get you to the Mylyn views. If you do not see Mylyn listed in the Show View dialog, then most likely you are running on an older version of Java. You'll need to install JDK 1.5.x or higher for Mylyn to work.
  • Why do artifact attributes still show up in my Project Tracker Mylyn editor after the project admin hides or removes them?
    This info gets cached in the Mylyn client. To clean this cache up and re-pull the info from the server, select the task that is showing the old attribute and delete it from the Mylyn task list. If this task is part of a query, you will notice that it is still available (just the cache was deleted). Regardless, next time you access this query it will show the correct attributes.
  • When I press the Validate button I get this message: "Your project is valid, but this server might not be compatible with your version of mylyn. You are running CollabNet API version 1.2.0. CollabNet server is running 1.1.0". What does it mean?
    You will get this message if your server is not running CEE 5.0 or greater. It simply means that some features will not be available. For example, you must be on CEE 5.0 for required fields to be validated in the rich editor. If you are on an earlier version, they are validated on the server and an error message is returned. Essentially, the user interface is cleaner when the newer version is used.
  • Where do I configure my client proxy settings?
    Configure proxy settings in the servers file (created when Subversion is installed on your system).
  • Does CollabNet Desktop - Eclipse Edition work with MyEclipse Enterprise Workbench 6.6.0?
    CollabNet Desktop - Eclipse Edition is designed to work with any Eclipse-based product. Make sure to install the version indicated for your version of Eclipse.
  • Does the CollabNet Desktop for Eclipse support client certificates for authorization to TeamForge?
    Yes. The CollabNet Desktop (Development Build 2.3.3 and higher) supports login using a certificate file (PKCS12) or smartcard (PKCS11).
  • Where can I find more information about the changes in the release?
    See the New and Noteworthy and Known Issues documents included with your update to review release changes.
