Customize Subversion repository URLs

The Desktop lets you select the repositories you work with frequently, customize the Subversion URLs for individual repositories, and add more than one root URL for a repository.

  1. In the CollabNet Sites view, right-click the Source Code node for a TeamForge project and select Repository preferences.
  2. In the Repository Preferences dialog, select the repositories you want displayed in the CollabNet Sites view.

    With TeamForge 6.1, Subversion repositories can be replicated. So you may want to check for and select a replica located nearby.

  3. To replace the default root URL with an alternative value for an individual project repository, select that row in the table and click Change Root URL(s).
    1. Select Custom.
    2. Click Add and enter a root URL value.

      You can define multiple root URLs for the repository. When the repository node is expanded in the CollabNet Sites view, the corresponding folders will be displayed under it. This allows project members to see branches for which they have path-based permissions to view, even if they do not have permission to a parent folder.

    3. Click Finish.
  4. Click OK in the Repository Preferences dialog.
    Tip: To add an alternate root URL or multiple URLs for a project repository, you can also use the Change SVN repository URL option from the repository's context menu.
