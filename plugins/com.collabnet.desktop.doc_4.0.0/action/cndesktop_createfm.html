Create a field mapping from a template

You set up field mappings to define how the data in one connector participant is transformed to the other.

There are three kinds of field mappings:
  • CUSTOM_XSLT - non-graphical mapping based on a user-supplied XSL transformation. It offers the greatest flexibility but requires programming skills to modify.
  • MAPFORCE - graphical mapping defined using Altova MapForce. See the CCF documentation on graphical mapping for details.
  • MAPPING_RULES - transformation rules set up using the custom field mapping editor.
  1. In the CollabNet Sites view, expand a node under Repository Mappings.
  2. Right-click a repository mapping direction and select New > Field Mapping . Here's an example.
  3. In the Create Field Mapping dialog, select a field mapping template.
  4. If you select Link field mapping to template, changes to the template will affect this field mapping.
  5. If you select Link field mapping to template, changes to the template will affect the repository mapping direction. The alternative is to define your own field mapping based on a template -- changes to the template will not affect the repository mapping direction.
  6. Click Finish.
