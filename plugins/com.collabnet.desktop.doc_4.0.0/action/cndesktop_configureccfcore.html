Configure connector properties

Before the connector can start synchronizing data, you must perform an initial configuration of properties such as system URLs, attachment size, timezones, and user credentials for the participating systems.

Note: To configure these properties, you need to be a TeamForge site administrator or have the requisite connector-related permissions in TeamForge.
  • In the CollabNet Sites view, right-click the connector node and select Edit connector.
  • In the Connector Properties screen:
    1. Test the connection to the central CCF database.
    2. Specify whether synchronization in a particular direction should happen on starting the connector
    3. Edit the default log template. For example, if you need more information about failed shipments, click Insert and add the variables you're interested in.
  • In the TeamForge Properties screen:
    1. Specify the URL of the TeamForge site. For example, http://myforge.mycompany.net.
    2. Specify the timezone and maximum size for attachments.
    3. Provide the connector username and password.
      Note: This account refers to the dedicated connector user in TeamForge who has the requisite permissions to access (read/write) the tracker involved in the mapping. It is important that no other user be allowed to log into the system with the connector user's username and modify this tracker's artifacts.

Depending on your connector, you'll see either a Quality Center Properties screen or a ScrumWorks Pro Properties screen.

  • In the Quality Center Properties screen:
    1. Enter the URL of the Quality Center server. For example, http://1.2.3.4:8080/qcbin.
    2. Specify the timezone and maximum size for attachments.
    3. Specify the Quality Center username and password.
      Note: The Quality Center user needs TDAdmin permissions.
  • In the ScrumWorks Pro Properties screen:
    1. Specify the URL of the ScrumWorks Pro WSDL. For example, http://localhost:8080/scrumworks-api/api2/scrumworks?wsdl or https://yourcompany.collab.net/scrumworks-api/api2/scrumworks?wsdl
    2. Specify the timezone and maximum size for attachments.
    3. Specify the ScrumWorks Pro username and password.
      Note: In order to avoid permissions conflicts in accessing ScrumWorks Products via the TeamForge connector, the user specified here should have Global Administrator privileges in ScrumWorks Pro. This user account is the dedicated connector user for ScrumWorks. It is important that no other user be allowed to login into the system with this connector user's username and modify data.
    4. Supply the Resync user's credentials.
      Note: This is the dedicated Resync user account -- it's different from the ordinary connector user account.
  • Save your changes.
