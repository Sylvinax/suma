Work with TeamForge connectors

A CollabNet TeamForge connector is an integration between TeamForge and another ALM product such as HP Quality Center, ScrumWorks Pro or MS Team Foundation Server, built using the CollabNet Connector Framework (CCF). The integration enables team members to work with either product and seamlessly see the latest changes made in the other one.

CollabNet Desktop - Eclipse Edition is one of several clients that the connector supports for remote administration.

The TeamForge connector is also referred to as CollabNet Connect / sync.

  • Get TeamForge connector functionality in your Desktop
    To get TeamForge connector functionality in your Desktop, you need to install the CollabNet Connector Framework components in addition to the CollabNet Desktop component.
  • What connector functionality can you access in your Desktop?
    The connector functionality that's available to you depends on the permissions you've been assigned in the TeamForge project involved in the integration.
  • Add a TeamForge connector
    To add a TeamForge connector site to your Desktop, you provide the URL of the connector server, and your account on the TeamForge instance for which the connector server is configured.
  • Configure connector properties
    Before the connector can start synchronizing data, you must perform an initial configuration of properties such as system URLs, attachment size, timezones, and user credentials for the participating systems.
  • HP Quality Center connector
    With the TeamForge - Quality Center connector, you can synchronize HP ALM defects and requirements with TeamForge tracker artifacts and planning folders in both directions.
  • ScrumWorks Pro connector
    With the TeamForge - ScrumWorks Pro connector, you can synchronize ScrumWorks Pro products with TeamForge projects in both directions.
  • Synchronize artifacts
    By default, synchronization is paused while you set up repository mappings. You need to explicitly resume synchronization in order to start shipping artifacts.
  • View log files
    The connector generates log files while synchronizing items. To get an overview of the artifacts synchronized, look at ccf-info.log. For detailed troubleshooting output, see ccf-debug.log.
  • Examine identity mappings
    Entries in the identity mapping table are generated as the connector synchronizes artifacts in the participating systems. Under normal operating conditions you may only need to view identity mappings. Sometimes, however, you may need to delete or modify a mapping.
  • Manage failed connector shipments
    When an artifact fails to ship, it is stored in the Failed Connector Shipments table. You can examine it, correct the problem, and replay it. Or you can fix the problem and re-run the connector. When the artifact ships successfully, you can mark older versions as fixed.
  • Monitor connector status
    While operating the connector, you can check the status, refresh details, and stop or start the connector.
